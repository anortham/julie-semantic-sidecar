# v0.1.0-rc.4

RC4 is the current source and package candidate for the Rust semantic sidecar. Preparing these
notes does not publish, tag, or release it.

## Changes

- Adds the `apple-x64-metal-portable` package profile for `x86_64-apple-darwin`.
- Configures GitHub's `macos-15-intel` runner to build and artifact-validate the Apple x64 profile.
- Keeps Metal built into both Apple executables with truthful ready-CPU fallback.
- Strengthens package-profile, workflow-matrix, backend-identity, throughput, and concurrent-process
  evidence contracts.
- Builds packages from a deterministic Cargo-vendored dependency tree and applies a strict,
  content-identity-bound native patch that replaces llama.cpp's three undefined Vulkan
  division-by-zero infinity expressions with bit-exact IEEE-754 values before shader compilation.
- Keeps `bge-small-en-v1.5-f32` as the standalone default and preserves the frozen
  `julie.embedding.sidecar` v1 wire protocol.

## Evidence boundary

- The exact released RC3 Apple arm64 archive has physical Apple Silicon CPU and Metal evidence.
  That checksum-bound result remains historical RC3 evidence and does not transfer to RC4.
- Apple x64 is a package candidate. A hosted Intel runner proves compilation, packaging, manifest
  validation, and unpacked artifact behavior; it is not physical Intel-Mac support evidence.
- Linux Vulkan, Windows Vulkan, and CUDA are package candidates in this release. These notes make no
  physical-hardware support claim for those lanes.
- Apple x64 requires exact-archive golden-vector conformance, Metal selection, CPU fallback,
  concurrency, determinism, and performance proof on a physical Intel Mac before support promotion.

## Approval boundary

The reviewed source must be committed before candidate archives are built. Pushing, workflow
dispatch, tagging, public release creation, and consumer pin updates remain separately approval
gated.
