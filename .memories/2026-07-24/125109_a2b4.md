## Fixed Windows package nondeterminism

- **WHAT:** Added deterministic MSVC linking for every Rust and CMake-produced PE, plus a native PowerShell environment self-test wired into the Windows release lane.
- **WHY:** RC4 cannot claim reproducible portable artifacts while identical source and toolchain builds produce different Windows bytes.
- **HOW:** `LINK`, Rust linker flags, and CMake EXE/shared/module linker flags now receive case-insensitive `/Brepro` handling without dropping caller flags. The PowerShell self-test covers empty, pre-seeded, encoded Rust, and case-variant environments before packaging starts.
- **IMPACT:** RC4 publication can resume after two independent Windows CI packages reproduce the same checksum. Fresh Claude review approved the final fix with zero findings.
