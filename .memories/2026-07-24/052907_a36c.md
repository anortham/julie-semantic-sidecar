## WHAT

Added the fourth portable sidecar package profile, `apple-x64-metal-portable`, across manifest validation, Bash and PowerShell package adapters, CI, the approval-gated candidate workflow, public docs, and promotion rules.

## WHY

Miller cannot complete its four-platform packaging story without an `x86_64-apple-darwin` sidecar artifact. The profile must use Metal when selected while remaining ready on CPU when Metal is unavailable.

## HOW

- Used the existing built-in `metal` feature and runtime backend selection; no wire or engine changes.
- Added a native-host-only package profile and `macos-15-intel` artifact-validation lanes.
- Bound manual candidate selection to the existing exact-checksum path.
- Added TDD contracts spanning validator, packagers, workflow matrices, README, and promotion docs.
- Documented that hosted Intel artifact validation is not physical Intel support evidence.

## IMPACT

The source is ready for review and an RC4 candidate build. Apple x64 support still requires a public exact-checksum asset and physical Intel-Mac conformance, Metal selection, CPU fallback, and performance proof.
